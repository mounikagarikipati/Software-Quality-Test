QA Arcade: Decision Tables & Pairwise Testing
We are going to refactor the existing "Password Bug Hunt" into a multi-module QA Arcade. The user will start at a Main Menu and can select which testing concept they want to practice.

[NEW] src/components/PasswordGame.js
Move the entire existing logic from App.js (including Luffy/Kaido images, password validation logic, etc.) into this dedicated component so it doesn't clutter the main file.
2. The Decision Table Game
To teach Decision Tables and Pairwise Testing, we need an environment with multiple complex rules that interact with each other.

[NEW] src/components/DecisionGame.js
The Concept: "Sanji's Restaurant Bill Calculator" (Keeping the One Piece vibe). A decision table requires evaluating combinations of inputs to determine an output.

Inputs to toggle:
Customer Type: (Marine, Pirate, Civilian)
Is Starving: (Yes / No)
Day of Week: (Weekday / Weekend)
Expected System: Perfect logic calculating a discount (e.g. "If Starving AND Pirate -> Free Food. If Marine AND Weekend -> Double Price.")
Buggy System: Has a hidden flaw. For example, the system forgets to charge Marines double on weekends.
The Gameplay: The player has to select different combinations from the dropdowns/checkboxes until they find an output where the Expected System differs from the Buggy System!
